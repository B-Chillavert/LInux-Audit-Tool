# Automated Linux System Audit & Hardening Tool

## Project Overview
This project is a lightweight, automated systems administration and defensive security tool written in native **Bash Scripting (`.sh`)**. It is designed to quickly audit a Linux operating system for common security misconfigurations, unauthorized user access, and perimeter exposure. 

By automating these checks, security analysts can rapidly baseline a system's hardening compliance status.

## Audit Capabilities & Findings Breakdown
During local deployment on a sandbox Debian/Kali Linux machine, the script successfully evaluated the following defensive areas:

1. **Privilege Layer Verification:** Confirmed the execution profile of the running user account to determine administrative scope.
2. **Active Session Tracking:** Monitored real-time user terminal attachments and active process footprints to spot unauthorized remote connections.
3. **UID 0 Persistence Scan:** Parsed the `/etc/passwd` configuration registry using an `awk` pattern-matching algorithm to identify all accounts with root-level supremacy (User ID 0), checking for malicious backdoor profiles.
4. **World-Writable Permission Audit:** Scanned the critical `/etc` configuration tree for globally editable file states (`perm -o+w`) that could allow low-privilege actors to manipulate system settings. *Result: Pass (0 files exposed).*
5. **Firewall Perimeter Validation:** Verified the status of the native host-based perimeter defense utilities. *Result: Alert Raised (UFW utility missing from host environment).*
